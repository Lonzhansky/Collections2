
Lambda calculus (Лямбда-числення)

Як правило, функціональні мови програмування реалізують
лямбда-числення. Оскільки лямбда-числення зосереджується на
композиції функцій, мови функціонального програмування
забезпечують виразні способи створення програмного забезпечення
у композиції функцій.


Лямбда-вирази - реалізація функціонального інтерфейсу.

Функціональний інтерфейс — інтерфейс лише з одним абстрактним
методом (sіngle abstract method, SAM).

Лямбда-вираз - екземпляр функціонального інтерфейсу.

Лямбда-вираз реалізує єдиний абстрактний метод/функцію
функціонального інтерфейсу.


Синтаксис лямбда-виразу
-----------------------------
Щоб створити лямбда-вираз, вказуємо вхідні
параметри (якщо вони є) у лівій частині лямбда-оператора
( -> ) і поміщаємо вираз або блок операторів у правій
частини лямбда-оператора

(перелік параметрів) -> {тіло функції}

Наприклад, лямбда-вираз
(a, b) -> a + b
вказує, що лямбда-вираз приймає
два аргументи a і b і повертає їхню суму.


Лямбда-вираз vs методи в Java
----------------------------------
Метод(функція) має: ім'я, список параметрів,
тіло, тип повернення.

Лямбда-вираз має ТІЛЬКИ тіло та список параметрів.
Без імені – функція анонімна, тому немає справи до
імені.
Тіло – це основна частина функції.
Немає типу повернення - компілятор Java може
визначити тип значення, що повертається, перевіривши
код. Не треба згадувати тип явно.


Щоб використовувати лямбда-вираз, потрібно або
створити свій власний функціональний інтерфейс,
або використовувати певний функціональний інтерфейс,
Java, наприклад, Runnable.

Interface Runnable
https://docs.oracle.com/javase/8/docs/api/java/lang/Runnable.html
Інтерфейс має єдиний метод run().
