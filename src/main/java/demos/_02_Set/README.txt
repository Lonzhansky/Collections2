
Interface Set<E> - набір (колекція), який не може містити елементи,
що повторюються. Це лінійна колекція об'єктів без дублікатів.
E - тип елементів, що підтримуються цим набором.
https://docs.oracle.com/javase/8/docs/api/java/util/Set.html

Interface Set моделює математичну абстракцію набору
та використовується для представлення наборів,
таких як курси, що становлять розклад студента,
або процеси, що виконуються на машині.


Популярні реалізації
------------------------
Class HashSet, Class LinkedHashSet, Class TreeSet


Внутрішня структура
----------------------
Class HashSet: усередині використовує HashMap для зберігання елементів.
Class LinkedHashSet: всередині використовує LinkedHashMap для зберігання
елементів.
Class TreeSet: усередині використовує TreeMap для зберігання елементів.


null елементи
---------------
Class HashSet: максимум один елемент null.
Class LinkedHashSet: максимум один елемент null.
Class TreeSet: не допускає жодного елемента null.


Елементи, що дублюються
-------------------------
Class HashSet: не дозволено
Class LinkedHashSet: не дозволено
Class TreeSet: не дозволено


Коли використовувати?
------------------------
Class HashSet: коли потрібні лише унікальні елементи
без будь-якого порядку.
Class LinkedHashSet: якщо потрібні лише унікальні елементи
в порядку вставки.
Class TreeSet: коли потрібні лише унікальні елементи
у певному порядку.
