
Interface Collector<T,A,R>
https://docs.oracle.com/javase/8/docs/api/java/util/stream/Collector.html

Параметри типу:
T – тип вхідних елементів операції приведення.
A - тип накопичення операції редукції, що змінюється.
R – тип результату операції редукції.

Операція редукції, що змінюється (mutable reduction
operation), яка накопичує вхідні елементи
у змінний контейнер результатів, додатково
перетворюючи накопичений результат на остаточне
представлення після обробки всіх вхідних елементів.

* Редукційні операції
https://docs.oracle.com/javase/8/docs/api/java/util/stream/package-summary.html#Reduction
Операція редукції (також звана згорткою/згинанням (fold))
бере послідовність вхідних елементів та об'єднує їх
в один підсумковий результат шляхом багаторазового застосування
операції об'єднання, наприклад, знаходження суми чи максимуму
набору чисел або накопичення елементів у списку.

Операції редукції можуть виконуватися як послідовно,
так і паралельно.


Class Collectors
https://docs.oracle.com/javase/8/docs/api/java/util/stream/Collectors.html

Реалізації Interface Collector, що реалізують
різні корисні операції редукції, такі
як накопичення елементів у колекції, підсумовування
елементів за різними критеріями тощо.
