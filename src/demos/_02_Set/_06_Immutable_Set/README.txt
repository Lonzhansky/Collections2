
Незмінний об’єкт не змінить свій внутрішній стан після того,
як ми його створимо. Це робить його потокобезпечним за замовчуванням.
Така ж логіка застосовна до незмінних наборів (Set).

Наприклад, якщо є екземпляр HashSet з деякими значеннями.
Зробивши його незмінним, ми створимо версію нашого набору
«лише для читання». Таким чином, будь-яка спроба змінити його стан
викличе UnsupportedOperationException.

Отже, навіщо нам це потрібно?

Найпоширенішим випадком використання незмінного набору є
багатопотокове середовище. Отже, можна обмінюватися незмінними
даними між потоками, не турбуючись про синхронізацію.

Водночас слід пам’ятати про важливий момент: незмінність стосується
лише множини, а не її елементів. Крім того, ми можемо змінювати
посилання на екземпляри елементів набору без проблем.
