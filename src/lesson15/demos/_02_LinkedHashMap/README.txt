
Class LinkedHashMap<K,V> - реалізація хеш-таблиці та пов’язаного
списку для Interface Map із передбачуваним порядком ітерацій.
Class LinkedHashMap розширює Class HashMap та відрізняється від
Class HashMap тим, що підтримує подвійний зв’язаний список, який
проходить через усі його записи (елементи).
Цей зв’язаний список визначає порядок ітерацій, який зазвичай є
порядком, у якому ключі були вставлені в Map (порядок вставки).
K - тип підтримуваних ключів.
V - тип зіставлених/відображених значень.
https://docs.oracle.com/javase/8/docs/api/java/util/LinkedHashMap.html


Class LinkedHashMap:
- Містить значення, що базуються на ключі.
- Містить унікальні елементи.
- Може мати один null ключ та кілька null значень.
- Несинхронізований. Кілька потоків можуть одночасно
модифікувати об'єкт.
- Підтримує порядок вставки.


Class LinkedHashMap може трохи поступатися за продуктивністю
батьківському Class HashMap, при цьому час виконання операцій
add(), contains(), remove() залишається константою - O(1).
Class LinkedHashMap потрібно трохи більше місця в пам'яті для
зберігання елементів та їх зв'язків.
